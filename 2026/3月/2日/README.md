## 1. 리포지토리 (repository)

코드와 변경 이력 전체를 저장하는 공간.  
단순한 파일 저장이 아니라 **누가, 언제, 무엇을 바꿨는지**까지 기록된다.

コードと変更履歴(へんこうりれき)の全(すべ)てを保存(ほぞん)する空間(くうかん)。  
単純(たんじゅん)なファイル保存(ほぞん)ではなく、**誰(だれ)が、いつ、何(なに)を変(か)えたか**まで記録(きろく)される。

```
Google Drive  → 최신 파일만 저장
Repository    → 1번 수정, 2번 수정, 3번 수정... 모든 역사를 저장
                덕분에 실수로 코드를 망쳐도 과거로 되돌릴 수 있다!
```

---

## 2. 리모트 / 로컬 리포지토리

### 리모트 리포지토리 (remote repository)

GitHub 같은 **인터넷 서버** 위에 있는 리포지토리.  
여러 사람이 함께 접근할 수 있는 **공유 저장소**.

GitHubのような**インターネットサーバー**上(じょう)にあるリポジトリ。  
いろんな人(ひと)が一緒(いっしょ)にアクセスできる**共有(きょうゆう)の保存場所(ほぞんばしょ)**。

```
팀원 A가 올린 코드 → 리모트 리포지토리 → 팀원 B가 받아서 작업
```

### 로컬 리포지토리 (local repository)

**내 컴퓨터 안**에 있는 리포지토리.  
인터넷 없이도 대부분의 작업이 가능하다.  
단, 변경사항을 팀원과 공유하려면 리모트 리포지토리에 `push` 해야 한다.

**自分(じぶん)のパソコンの中(なか)**にあるリポジトリ。  
インターネットなしでもほとんどの作業(さぎょう)が可能(かのう)。

```
로컬에서 작업 → git push → 리모트(GitHub)에 반영
```

## 3. 브랜치 (branch)

코드의 분기점. 나무의 가지처럼 메인 브랜치에서 뻗어나와  
**독립적으로 작업할 수 있는 공간**.

コードの分岐点(ぶんきてん)。木(き)の枝(えだ)のようにメインブランチから伸(の)びて、  
**独立的(どくりつてき)に作業(さぎょう)できる空間(くうかん)**。

```
main ─────────────────────────────► (실제 서비스)
       │
       └── feature-login ──────────► 로그인 기능 개발
       │
       └── feature-comment ────────► 댓글 기능 개발

각 브랜치에서 안전하게 작업 → 완성되면 main에 합침(merge)
```

---

## 4. 리모트 / 로컬 브랜치

### 리모트 브랜치 (remote branch)

리모트 리포지토리(GitHub)에 존재하는 브랜치.  
로컬에서 `git push`를 하면 로컬 브랜치의 내용이 리모트 브랜치에 반영된다.  
→ **GitHub에서 눈으로 확인할 수 있는 브랜치**

リモートリポジトリ(GitHub)にあるブランチ。  
ローカルで`git push`をするとローカルブランチの内容(ないよう)がリモートブランチに反映(はんえい)される。

### 로컬 브랜치 (local branch)

내 컴퓨터에만 존재하는 브랜치.  
`push`하기 전까지는 **다른 사람에게 보이지 않는다**.

自分(じぶん)のパソコンにだけ存在(そんざい)するブランチ。  
`push`する前(まえ)までは**他(ほか)の人(ひと)には見(み)えない**。

```bash
# 로컬에서 브랜치를 10개 만들어도
# push하지 않으면 GitHub에는 하나도 올라가지 않는다!
git branch feature-login    # 로컬에만 존재
git push origin feature-login  # 그제서야 GitHub에 반영
```

## 5. 디폴트 브랜치 (default branch)

리포지토리 생성 시 **자동으로 만들어지는 기준 브랜치**.  
보통 `main` 또는 `master`라는 이름.

リポジトリ作成時(さくせいじ)に**自動(じどう)で作(つく)られる基準(きじゅん)ブランチ**。  
普通(ふつう)`main`または`master`という名前(なまえ)。

```
역할:
✅ 모든 작업의 최종 목적지
✅ 실제 서비스에 배포되는 코드가 담겨 있음

규칙:
❌ 직접 작업하지 않는다
✅ 반드시 브랜치를 나눠서 작업 후 merge
```

## 6. 워크트리 (work tree)

내가 실제로 파일을 열고 편집하는 공간.  
VS Code나 터미널에서 보이는 파일들이 있는 **폴더 자체**.  
git은 이 워크트리의 변화를 감지해서 **"어떤 파일이 바뀌었는지"** 추적한다.

自分(じぶん)が実際(じっさい)にファイルを開(ひら)いて編集(へんしゅう)する空間(くうかん)。  
gitはこのワークツリーの変化(へんか)を感知(かんち)して**「どのファイルが変(か)わったか」**を追跡(ついせき)する。

---

## 7. 인덱스 / 스테이지 (index / stage)

워크트리와 커밋 사이의 **대기 공간**.  
파일을 수정했다고 바로 저장되는 게 아니라 **2단계 구조**로 확정된다.

ワークツリーとコミットの間(あいだ)の**待機空間(たいきくうかん)**。  
ファイルを修正(しゅうせい)してもすぐ保存(ほぞん)されるわけではなく**2段階(だんかい)の構造(こうぞう)**。

```bash
# 2단계 구조
git add 파일명       # 워크트리 → 스테이지 (올리기)
git commit -m "메시지"  # 스테이지 → 리포지토리 (최종 확정)
```

```
덕분에 수정한 파일 중 원하는 것만 골라서 커밋할 수 있다!

예시:
파일 3개 수정 (A.rb, B.rb, C.rb)
git add A.rb        # A만 스테이지에 올리기
git commit          # A만 커밋 → B, C는 다음 커밋에
```

## 8. 전체 흐름 한눈에 보기

```
[워크트리]         [스테이지]        [로컬 리포지토리]    [리모트 리포지토리]
파일 수정    →    git add    →      git commit     →      git push
(VS Code)                         (이력 저장)           (GitHub 반영)

                                                          ↓
                                                       git pull
                                                    (팀원 코드 받기)
```

## 🔑 한눈에 보는 키워드

| 용어              | 한국어                           | 日本語(にほんご)                                |
| :---------------- | :------------------------------- | :---------------------------------------------- |
| repository        | 코드와 변경 이력을 저장하는 공간 | コードと変更履歴を保存する空間                  |
| remote repository | GitHub 서버 위의 공유 저장소     | GitHubサーバー上の共有保存場所                  |
| local repository  | 내 컴퓨터 안의 저장소            | 自分のパソコンの中の保存場所                    |
| branch            | 독립 작업 공간 (나무 가지)       | 独立作業空間（木の枝）                          |
| default branch    | main/master — 최종 배포 브랜치   | main/master — 最終配布ブランチ                  |
| work tree         | 실제 파일을 편집하는 폴더        | 実際にファイルを編集するフォルダ                |
| index / stage     | git add 후 커밋 전 대기 공간     | git add後、コミット前の待機空間                 |
| `git add`         | 워크트리 → 스테이지              | ワークツリー → ステージ                         |
| `git commit`      | 스테이지 → 로컬 리포지토리       | ステージ → ローカルリポジトリ                   |
| `git push`        | 로컬 → 리모트(GitHub)            | ローカル → リモート(GitHub)                     |
| `git pull`        | 리모트 → 로컬 (팀원 코드 받기)   | リモート → ローカル（チームのコードを受け取る） |
