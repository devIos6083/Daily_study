## 1. .gitignore란?

git이 **추적하면 안 되는 파일이나 폴더를 지정**하는 설정 파일.  
`.gitignore`에 작성한 파일은 `git add`를 해도 스테이지에 올라가지 않고,  
`git status`에도 표시되지 않는다.

gitが**追跡(ついせき)するべきでないファイルやフォルダを指定(してい)**する設定(せってい)ファイル。

### 왜 필요한가?

```
🔴 보안 위협
  .env (API 키, 비밀번호, 환경변수)
  → 유출되면 큰 사고!

🟡 불필요한 파일
  .DS_Store (Mac 자동 생성)
  .idea/, .vscode/ (IDE 설정 파일)
  → 개인 환경 파일이라 팀원과 공유할 필요 없음

🟠 자동 생성 파일
  node_modules/, vendor/
  → 용량이 크고 npm install로 언제든 재생성 가능

⚪ 빌드 결과물
  dist/, build/
  → 소스코드에서 자동으로 생성됨
```

### 작성 방법

```gitignore
# 특정 파일 무시
.env
.DS_Store

# 특정 확장자 무시
*.log
*.tmp

# 특정 폴더 무시
node_modules/
build/

# 예외 (무시하지 않음)
!important.log
```

### 주의사항

```bash
# ❌ 이미 git이 추적 중인 파일은 .gitignore에 추가해도 무시되지 않는다!
# ✅ 먼저 추적을 해제해야 한다
git rm --cached 파일명
```

### 흐름

```
워크트리의 파일
    ↓
.gitignore가 필터링
    ↓
해당하지 않는 파일만 git add 가능
```

## 2. 터미널(Terminal)이란?

사용자가 컴퓨터에 **텍스트 명령어를 입력해서 결과를 확인**하는 인터페이스.  
프로그램이 아니라 **"창구"의 역할**.

ユーザーがパソコンに**テキストの命令語(めいれいご)を入力(にゅうりょく)して結果(けっか)を確認(かくにん)**するインターフェース。  
プログラムではなく**「窓口(まどぐち)」の役割(やくわり)**。

```
사용자 → 터미널 → 셸 → OS → 결과 반환
ユーザー → ターミナル → シェル → OS → 結果返還(けっかへんかん)
```

| Mac 터미널 앱 | 특징                    |
| :------------ | :---------------------- |
| Terminal.app  | macOS 기본 제공         |
| iTerm2        | 많이 쓰이는 서드파티 앱 |

## 3. 셸(Shell)이란?

터미널에서 입력한 명령어를 해석해서 **OS에 전달하는 프로그램**.  
터미널이 "창구"라면 셸은 **"통역사"**.

ターミナルで入力(にゅうりょく)した命令語(めいれいご)を解釈(かいしゃく)してOSに伝(つた)える**「通訳者(つうやくしゃ)」**。

| 셸 종류 | 특징                                |
| :------ | :---------------------------------- |
| bash    | Linux 기본 셸                       |
| zsh     | macOS 기본 셸 (macOS Catalina 이후) |
| fish    | 자동 완성이 강력한 셸               |

```bash
# 현재 사용 중인 셸 확인
echo $SHELL
# → /bin/zsh
```

## 4. 파일 경로(Path)

파일이나 폴더의 위치를 나타내는 **주소**. 두 종류가 있다.

ファイルやフォルダの位置(いち)を表(あらわ)す**住所(じゅうしょ)**。

### 절대 경로 (Absolute Path)

루트(`/`)에서 시작하는 **전체 경로**.

```bash
/Users/hongkyu/bootcamp/git/prerequisite-knowledge.txt
```

### 상대 경로 (Relative Path)

**현재 위치를 기준**으로 한 경로.

```bash
# 현재 위치가 bootcamp/ 일 때
git/prerequisite-knowledge.txt

# 한 단계 위로
../other-folder/file.txt
```

## 5. 현재 디렉토리 (Current Directory)

지금 터미널에서 **작업 중인 위치(폴더)**.  
"현재 파일"이 아니라 **"지금 내가 서 있는 폴더"**라고 이해하면 된다.

今(いま)ターミナルで**作業中(さぎょうちゅう)の位置(いち)(フォルダ)**。

```bash
pwd       # 현재 경로 확인 (Print Working Directory)
cd 폴더명  # 디렉토리 이동 (Change Directory)
ls        # 현재 디렉토리 내용 확인 (List)
```

터미널 프롬프트에도 현재 디렉토리가 표시된다:

```
hongkyu-kang@MacBookAir bootcamp %
                         ↑ 현재 디렉토리
```

## 6. 터미널 단축키

| 단축키          | 동작                          |
| :-------------- | :---------------------------- |
| `↑` (위 화살표) | 이전에 입력한 명령어 불러오기 |
| `tab`           | 명령어/파일명 자동 완성       |
| `ctrl + c`      | 실행 중인 명령어 강제 종료    |
| `ctrl + l`      | 터미널 화면 지우기            |
| `ctrl + a`      | 커서를 줄 맨 앞으로 이동      |
| `ctrl + e`      | 커서를 줄 맨 끝으로 이동      |

## 7. echo 명령어

터미널에서 **문자열을 출력**하는 명령어.

ターミナルで**文字列(もじれつ)を出力(しゅつりょく)**する命令語(めいれいご)。

```bash
# 기본 출력
echo "Hello, World!"

# 변수 사용
name="カンホンギュ"
echo "私の名前は${name}です"
# → 私の名前はカンホンギュです

# 줄바꿈 없이 출력
echo -n "ローディング中"
echo "...完了"
# → ローディング中...完了

# 이스케이프 문자 사용 (-e 옵션)
echo -e "一行目\n二行目\n三行目"
# → 一行目
#   二行目
#   三行目

# 명령어 결과 안에 포함시키기 $()
echo "現在のディレクトリ: $(pwd)"
echo "現在の日時: $(date)"
```

| 키워드            | 설명                                     |
| :---------------- | :--------------------------------------- |
| `.gitignore`      | git 추적에서 제외할 파일/폴더 지정       |
| `git rm --cached` | 이미 추적 중인 파일의 추적 해제          |

| `echo $SHELL`     | 현재 사용 중인 셸 확인                   |
| 절대 경로         | `/`에서 시작하는 전체 경로               |

| `cd`              | 디렉토리 이동                            |
| `ls`              | 현재 디렉토리 내용 확인                  |
| `echo`            | 문자열 출력                              |
| `$(명령어)`       | 명령어 결과를 문자열 안에 포함           |
| `ctrl + c`        | 실행 중인 명령어 강제 종료               |
