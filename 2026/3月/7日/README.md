## 1. 파이프(Pipe)란?

**앞 명령어의 출력 결과를 뒷 명령어의 입력으로 연결하는 기능**.  
`|` (수직 막대) 기호를 사용하며, 여러 개를 연결해서 **데이터를 단계적으로 가공**할 수 있다.  
각 명령어는 독립적으로 동작하지만, 파이프로 연결하면 하나의 흐름처럼 작동한다.

**前(まえ)のコマンドの出力結果(しゅつりょくけっか)を次(つぎ)のコマンドの入力(にゅうりょく)として渡(わた)す機能(きのう)**。  
`|`(パイプ)記号(きごう)を使(つか)い、複数(ふくすう)をつなげてデータを段階的(だんかいてき)に加工(かこう)できる。

```
명령어A | 명령어B | 명령어C

명령어A 실행
    ↓ 출력 결과를 파이프로 전달
명령어B가 받아서 처리
    ↓ 그 결과를 다시 파이프로 전달
명령어C가 받아서 최종 처리
```

```bash
# 예시: sh 파일 목록 조회
ls | grep "sh"
# ls → 현재 폴더의 모든 파일 목록 출력
# grep "sh" → 그 중에서 "sh"가 포함된 줄만 필터링
```

## 2. 자주 쓰는 파이프 조합

### grep — 특정 패턴 필터링

```bash
# grep: 입력에서 특정 문자열이 포함된 줄만 골라냄
# "sh"가 포함된 파일 이름만 출력
ls | grep "sh"

# PATH 관련 환경변수만 출력
env | grep "PATH"
```

### wc — 개수 세기

```bash
# wc -l: 줄 수(line count) 세기
cat pipe.sh | wc -l
# → pipe.sh 파일의 전체 줄 수

# wc -w: 단어 수(word count) 세기
echo "hello world foo bar" | wc -w
# → 4

# grep + wc 조합 — "sh" 포함 파일이 몇 개인지
ls | grep "sh" | wc -l
```

### sort — 정렬

```bash
# sort: 알파벳 순으로 오름차순 정렬
echo -e "banana\napple\norange\ngrape" | sort
# → apple, banana, grape, orange

# sort -r: 내림차순(reverse) 정렬
echo -e "banana\napple\norange\ngrape" | sort -r
# → orange, grape, banana, apple
```

### uniq — 중복 제거

```bash
# uniq: 연속된 중복 줄을 제거
# 주의: 정렬(sort) 후에 써야 제대로 동작함!
# (uniq는 바로 인접한 중복만 제거하기 때문)
echo -e "apple\napple\nbanana\nbanana\norange" | sort | uniq
# → apple, banana, orange
```

### tr — 문자 변환

```bash
# tr '변환전' '변환후': 문자를 다른 문자로 치환
# 소문자 → 대문자
echo "hello world" | tr 'a-z' 'A-Z'
# → HELLO WORLD

# 대문자 → 소문자
echo "HELLO WORLD" | tr 'A-Z' 'a-z'
# → hello world
```

### 여러 파이프 연결

```bash
# 파이프를 3개 이상 연결해서 단계별로 가공
# ① env로 환경변수 전체 출력
# ② PATH 관련만 필터링
# ③ 소문자를 대문자로 변환
env | grep "PATH" | tr 'a-z' 'A-Z'
```

## 3. 표준 입력 — read

**스크립트 실행 중에 사용자로부터 값을 직접 입력받는 방법**.  
`read 변수명` 으로 입력을 기다리다가, 사용자가 Enter를 누르면 값을 변수에 저장한다.  
대화형(interactive) 스크립트를 만들 때 필수적으로 사용한다.

**スクリプト実行中(じっこうちゅう)にユーザーから直接(ちょくせつ)値(あたい)を受(う)け取(と)る方法(ほうほう)**。  
`read 変数名(へんすうめい)`で入力(にゅうりょく)を待(ま)ち、Enterを押(お)すと変数(へんすう)に保存(ほぞん)される。

```bash
# 기본 사용법
echo "名前を入力してください:"
read name                          # 사용자가 입력할 때까지 대기
echo "こんにちは、${name}さん！"   # 입력받은 값 출력
```

### 여러 변수를 한 번에 입력받기

```bash
# 공백으로 구분해서 여러 변수에 동시 저장
echo "都市と年齢を入力してください:"
read city age
# 입력: 新潟 25
# → city = "新潟", age = "25" 로 각각 저장

echo "都市: ${city}"
echo "年齢: ${age}"
```

## 4. read 옵션

### `-p` — 프롬프트 한 줄로

```bash
# -p "메시지": echo 없이 입력 안내 메시지와 read를 한 줄로
read -p "好きな食べ物を入力してください: " food
echo "好きな食べ物: $food"

# echo + read 두 줄 쓰는 것과 동일하지만 더 간결함
```

### `-s` — 비밀 입력 (화면에 안 보임)

```bash
# -s (silent): 입력 내용을 화면에 표시하지 않음
# 비밀번호처럼 노출되면 안 되는 값을 입력받을 때 사용
read -sp "パスワードを入力してください: " password
echo ""            # -s 사용 후 줄바꿈이 없으므로 수동으로 추가
echo "パスワードを受け取りました。"
# 입력한 비밀번호는 $password에 저장되지만 화면에는 표시되지 않음
```

|     옵션      | 의미                     | 사용 예                |
| :-----------: | :----------------------- | :--------------------- |
| `-p "메시지"` | 입력 전 안내 메시지 표시 | 프롬프트를 한 줄로     |
|     `-s`      | 입력 내용을 화면에 숨김  | 비밀번호 입력          |
|     `-sp`     | 두 옵션 동시 적용        | 비밀번호 + 안내 메시지 |

## 5. 실전 예제

### 입력값으로 조건 분기

```bash
# 사용자가 입력한 값을 바로 조건문에 활용
read -p "年齢を入力してください: " input_age

if [ $input_age -ge 20 ]; then
  echo "成人です"
else
  echo "未成年です"
fi
```

### 파이프 + while read — 파일/데이터를 한 줄씩 처리

```bash
# 파이프로 넘어온 데이터를 while read로 한 줄씩 받아서 처리
# "파이프의 출력 → read의 입력"으로 연결되는 패턴
echo -e "Ruby\nShell\nJavascript\nTypeScript" | while read line; do
  echo "言語: $line"
done
# → 言語: Ruby
#   言語: Shell
#   言語: Javascript
#   言語: TypeScript
```

### 반복 입력으로 배열 만들기

```bash
# for 루프 안에서 read를 써서 여러 번 입력을 받아 배열에 쌓는 패턴
echo "果物を3つ入力してください"
fruits=()

for i in $(seq 1 3); do
  read -p "${i}つ目: " fruit   # 매 반복마다 하나씩 입력받음
  fruits+=("$fruit")           # 입력받은 값을 배열에 추가
done

echo "入力した果物: ${fruits[@]}"
```

## 🔑 한눈에 보는 키워드

| 키워드                      | 설명                                  |
| :-------------------------- | :------------------------------------ |
| `명령어A \| 명령어B`        | 파이프 — A의 출력을 B의 입력으로 연결 |
| `grep "패턴"`               | 패턴이 포함된 줄만 필터링             |
| `wc -l`                     | 줄 수 세기                            |
| `wc -w`                     | 단어 수 세기                          |
| `sort`                      | 오름차순 정렬                         |
| `sort -r`                   | 내림차순 정렬                         |
| `sort \| uniq`              | 중복 제거 (반드시 sort 먼저!)         |
| `tr 'a-z' 'A-Z'`            | 소문자 → 대문자 변환                  |
| `tr 'A-Z' 'a-z'`            | 대문자 → 소문자 변환                  |
| `read 변수명`               | 사용자 입력을 변수에 저장             |
| `read 변수1 변수2`          | 공백 구분으로 여러 변수에 동시 저장   |
| `read -p "메시지" 변수`     | 안내 메시지와 함께 입력받기           |
| `read -s 변수`              | 입력 내용 숨기기 (비밀번호용)         |
| `파이프 \| while read line` | 파이프 출력을 한 줄씩 처리            |
